# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in this repository, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Email: security@certifieddata.io

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact

We will acknowledge receipt within 48 hours and aim to provide a fix or mitigation within 14 days for critical issues.

## Scope

This repository contains public contract artifacts, SDKs, and a mock server. It does not contain:
- API keys or secret material
- Production authentication credentials
- Private signing keys
- Payment processor secrets

If you believe you have found a vulnerability in the live CertifiedData Payments API or platform, please report it via the email above.

## Safe harbor

We consider security research conducted in good faith under this policy to be authorized. We will not pursue legal action against researchers who:
- Report issues through the responsible disclosure process above
- Do not access, modify, or delete customer data
- Do not disrupt platform availability
